DNN Alpha Access Instructions

How to get Kovan Ether (KETH)

There are two ways for you to obtain Kovan Ether:

  1. Ask a member of the DNN team (Recommended)
  2. Use Parity

To ask for Kovan Ether from a member of the DNN team, sign in to the DNN slack and direct message either @dondrey or @samit with your public address. KETH will then be transfered directly to the address that you provide.

If you choose to obtain Kovan Ether through Parity, you will need to verify your Ethereum account using a small amount of real ether. Once verified, you will be able to obtain 5 Kovan Ether (KETH) every 24 hours. The following video will show you how to verify your account.

For a complete explaination of how to get KETH using Parity refer to the Kovan Testnet faucet repo.

How to get DNN tokens

To obtain DNN tokens use the faucet option located under Buy DNN tokens in your wallet.

Getting started with a Web3-Enabled Browser

Web3 browsers allow you to interact with decentralized applications (called DApps) on the Ethereum blockchain. Currently, the most popular available options are two different web3-enabled browsers and a web3-enabled plugin.

  1. MetaMask - Web3 Plugin
  2. Parity - Web3 Browser
  3. Mist - Web3 Browser

MetaMask - Setup Instructions

The MetaMask plugin allows you to interact with the Ethereum blockchain without the need for a seperate desktop application. Currently, MetaMask is supported in Google Chrome, Firefox, and Microsoft Edge. For this tutorial we'll assume you will be using Google Chrome, however, after the plugin is installed, setting up a wallet using MetaMask will be similar, if not the same between various browser implementations.

To install MetaMask on Google Chrome, head over to the Google Web Store, and download the MetaMask Plugin.

Read over the MetaMask Privacy Notice, then accept.

Read over the MetaMask Terms of Use, then accept.

At this point you will be prompted to create a password. Your password will be used to secure your ether and DNN tokens, so pick something strong!

Mark down the phrase listed in this step. This phrase will be used to recover your password should you forget it. WARNING: You will not be able to retrieve this again. If you lose it, you will never be able to access your ether or DNN tokens from this wallet.

Your account on MetaMask is now successfully created. MetaMask is currently connected to Mainnet, but to interact with DNN, MetaMask must be configured with the Kovan network. To change the network MetaMask is connected to, click on the diamond or text to the right of the diamond at the upper left corner to view a list of available network options.

Select Kovan Test Network.

To add additional accounts, click on the user icon with circular arrows. On this screen, you can select a different account or add a new one.

Occasionaly, MetaMask may lock your account for security reasons. If this happens, you will need to provide your MetaMask password to unlock it.

Refer to MetaMask's blog post or website for further information regarding setup.

Parity - Setup Instructions

To get started with Parity, you will need to download the Parity desktop application from Parity Technologies's site. Parity is currently available on Ubuntu, Mac OSX, Docker, and Windows.

The rest of the setup instructions will be based on the Ubuntu and Mac OSX version of Parity, however the steps to set up Parity will be largely the same between platforms.

To install Parity for Ubuntu or Mac OSX run the command in your terminal bash <(curl https://get.parity.io -Lk)

To test if Parity installed correctly run the following command in your terminal

parity --version

If Parity is properly installed, you should see the version of Parity that you are running neatly displayed beneath the command. If you do not, please refer to Parity's Wiki for troubleshooting help.

With Parity installed, you are ready to configure it to connect to the Kovan Testnet. By default, Parity connects to Mainnet, to modify this, you will need to run parity and then navigate to the config directory.

Run Parity with the following command to generate the configuration directory

parity

Once Parity generates a public address, quite the process by holding CTRL+C. Depending on what platform you are using the configuration directory will be different. The following a the directories in which you can find the Parity configuration file.

Windows: %UserProfile%\AppData\Roaming\Parity\Ethereum\ Linux: ~/.local/share/io.parity.ethereum/ Mac OSX: $HOME/Library/Application Support/io.parity.ethereum/

To let parity know you are running a configuration in a different directory, run Parity using the following command

parity --config path/to/config.toml

Refer to Parity's Wiki for more information about Parity Configuration

Once inside Parity directory, create the Parity configuration file, if one does not already exist

touch config.toml

Inside the config.toml file add or modify the the file to have the following lines

[parity]
chain = "kovan"

After you save your changes to the config.toml file, run Parity

parity

With Parity now running, open up your web browser and navigate to 127.0.0.1:8180 to access Parity's web interface.

Choose next to proceed past the welcome screen.

Read over Parity's Terms and Conditions, then accept them to continue.

Create a name and password for your account. Be sure to choose a strong password to ensure that your account is as secure as possible. Once you have finished entering a name and username, choose a profile picture, and click create to continue.

Mark down your newly created wallet address and recovery password phrase. Be sure not to lose your recovery password phrase, should you forget it, you will no longer have access to funds in your Ethereum account.

A setup completion screen will inform you that your Parity node has been successfully setup.

To use Parity from within a traditional browser like Chrome, install the Parity extension by clicking on INSTALL THE EXTENSION NOW! from the orange popup.

Click ADD TO CHROME to install the Parity extension.

At the top right corner of chrome you will see the Parity icon. Clicking on this icon will reveal the network Parity is connected to. To use DNN, Parity must be connected to the Kovan Testnet. and not Mainnet (contrary what is shown in the screenshot below).

Enabling Local Storage

Local storage gives web applications the ability to store data within your browser. This allows for a smoother experience because data does not have to be downloaded prior to viewing. In DNN's case, local storage is used to store the contents of articles that are retrieved from the DNN network.

Local Storage is supported by Google Chrome (version > 4), Internet Explorer (version >= 8), Edge, Firefox (version >= 3.5), Safari (version >= 4), and Opera (version >= 11.5)

Below of the steps to enable Local Storage in Google Chrome.

Click on the three vertical dots at the top right corner of chrome

Choose Settings.

Scroll down to the bottom until you see the Advanced option.

Click on the advanced option and then choose Content settings.

Choose Cookies.

Ensure that the option Allow sites to save and read cookie data is enabled.
